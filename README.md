# 📖 Introduction

AI Quiz Generator is an intelligent web application designed to automate the process of generating professional Multiple Choice Questions (MCQs) from educational content.

Instead of creating quiz questions manually, the system analyzes the provided text, understands its meaning using Artificial Intelligence, identifies important concepts, and generates meaningful assessment questions automatically.

The application combines multiple AI technologies including Natural Language Processing (NLP), Transformer Models, Sentence Embeddings, Machine Learning, and Semantic Similarity Analysis to produce context-aware quiz questions.

Unlike traditional keyword-based quiz generators, this system focuses on understanding the semantic meaning of educational content before generating questions. This approach significantly improves question quality, relevance, and readability.

The application supports both plain text and PDF documents, allowing users to upload learning material directly without additional preprocessing.

# 📥 Step 1 — Receiving User Input

The execution begins when a user submits educational content through the web interface.

The application currently supports two different input formats:

- Plain Text
- PDF Documents

If the uploaded content is a PDF file, the system automatically extracts readable text before continuing to the Artificial Intelligence pipeline.

All requests are handled through the FastAPI backend.

---

# 📄 Step 2 — PDF Processing

When a PDF document is uploaded, the application uses **PyMuPDF (fitz)** to extract text from every page.

Instead of converting pages into images, the library directly reads textual content from the document.

This approach provides:

- Faster processing
- Better text accuracy
- Lower memory usage
- Support for large educational documents

The extracted text is merged into a single continuous document before preprocessing begins.

---

# 🧹 Step 3 — Text Cleaning

Before any Deep Learning model receives the content, the text is cleaned and normalized.

This stage removes unnecessary information that could reduce question quality.

The preprocessing stage includes:

- Removing extra whitespace
- Cleaning unwanted symbols
- Formatting sentences
- Standardizing punctuation
- Preparing readable input for NLP models

Clean input significantly improves the overall performance of the downstream Transformer models.

---

# ✂️ Step 4 — Sentence Segmentation

After preprocessing, the document is divided into individual sentences using the Natural Language Toolkit (NLTK).

Sentence segmentation is important because the system processes educational concepts one sentence at a time.

Each sentence becomes an independent candidate for semantic analysis.

This enables the application to evaluate the importance of different concepts separately rather than treating the entire document as one large block of text.

---

# 🧠 Step 5 — Semantic Embedding Generation

Once the sentences have been extracted, they are converted into dense numerical vector representations using the Sentence Transformer model.

Model used:

```
all-MiniLM-L6-v2
```

Unlike traditional keyword-based approaches, sentence embeddings capture the semantic meaning of complete sentences.

Two sentences that express similar ideas produce similar vector representations even if they use different vocabulary.

This allows the application to understand concepts rather than individual words.

---

# 🎯 Step 6 — Semantic Importance Ranking

After generating embeddings, the system evaluates the importance of every sentence.

Instead of randomly selecting educational content, the AI identifies sentences that best represent the overall document.

This produces significantly higher quality quiz questions because the generated questions originate from meaningful educational concepts rather than isolated facts.

---

# 🗂️ Step 7 — Topic Clustering

The generated embeddings are grouped using the KMeans clustering algorithm.

Purpose:

- Detect multiple topics
- Avoid repetitive questions
- Improve topic diversity
- Increase quiz coverage

Rather than generating several questions from the same paragraph, clustering distributes questions across different concepts contained within the document.

---

# 🤖 Step 8 — Transformer Question Generation

This is the core Deep Learning component of the project.

The application uses the following pretrained Transformer model:

```
valhalla/t5-small-qg-hl
```

This model is specifically fine-tuned for Question Generation.

The selected answer span is highlighted before being passed to the Transformer.

The model then generates a natural language question whose answer corresponds to the highlighted text.

Unlike template-based generators, this stage performs genuine neural language generation.

If the dedicated Question Generation model is unavailable, the application automatically falls back to:

```
t5-small
```

This fallback mechanism ensures that the application remains operational even if the specialized model cannot be downloaded.

---

# 🎲 Step 9 — Distractor Generation

Generating incorrect answer options is often more difficult than generating the question itself.

The application uses multiple Natural Language Processing techniques to identify plausible distractors.

These include:

- Part-of-Speech Tagging
- WordNet
- Semantic Similarity
- Related Concepts

The goal is to produce answer options that are believable while remaining incorrect.

This improves the educational quality of the generated MCQs.

---

# 📊 Step 10 — Difficulty Prediction

Every generated question is passed to the custom PyTorch Difficulty Classification model.

Unlike rule-based systems, this classifier has been trained using thousands of labelled Multiple Choice Questions.

The neural network predicts one of three categories:

```
Easy
Medium
Hard
```

The classifier operates on Sentence Transformer embeddings rather than raw text.

This enables semantic understanding during prediction.

---

# 🧩 Neural Network Architecture

The custom classifier follows a feed-forward neural network architecture.

```
Sentence Embedding
       │
       ▼
Linear Layer (384 → 128)
       │
       ▼
ReLU Activation
       │
       ▼
Dropout
       │
       ▼
Linear Layer (128 → 64)
       │
       ▼
ReLU Activation
       │
       ▼
Dropout
       │
       ▼
Output Layer (64 → 3)
       │
       ▼
Easy / Medium / Hard
```

This lightweight architecture has been optimized for CPU execution while maintaining accurate predictions.

# 🧠 Transformer Question Generation Model

## Primary Model

```
valhalla/t5-small-qg-hl
```

The primary model responsible for question generation is **valhalla/t5-small-qg-hl**.

This model is based on Google's Text-to-Text Transfer Transformer (T5) architecture and has been specifically fine-tuned for Question Generation tasks.

Unlike traditional rule-based systems, the Transformer understands contextual information before generating a question.

Instead of selecting random keywords, the model analyzes the relationship between concepts, entities, and surrounding text before producing a grammatically correct and meaningful question.

### Responsibilities

- Context Understanding
- Question Generation
- Natural Language Processing
- Sequence-to-Sequence Learning
- Educational Question Creation

---

# 🔄 Fallback Model

```
t5-small
```

The application includes an automatic fallback mechanism.

If the primary Question Generation model cannot be loaded, the system automatically switches to the standard **t5-small** model.

This ensures that quiz generation continues even when specialized resources are temporarily unavailable.

Benefits include:

- Improved reliability
- Continuous service availability
- Better deployment stability
- Reduced runtime failures

---

# 🧩 Sentence Embedding Model

## Model Used

```
all-MiniLM-L6-v2
```

Semantic understanding is performed using the Sentence Transformer model **all-MiniLM-L6-v2**.

Rather than comparing individual words, this model converts complete sentences into dense numerical vectors known as embeddings.

These embeddings represent the semantic meaning of text.

Two sentences discussing the same concept will produce similar embeddings even if different vocabulary is used.

This enables the application to understand educational content much more effectively than keyword-based systems.

### Primary Responsibilities

- Sentence Embedding
- Semantic Similarity
- Context Understanding
- Information Retrieval
- Topic Identification
- Educational Content Analysis

---

# 📊 Embedding Dimension

The selected Sentence Transformer generates a **384-dimensional embedding vector** for every sentence.

Example

```
Educational Sentence

↓

Sentence Transformer

↓

[384 Numerical Values]

↓

Semantic Representation
```

These vectors become the foundation for several Artificial Intelligence operations inside the project.

---

# 🎯 Semantic Similarity

Semantic Similarity determines how closely two sentences are related in meaning.

Instead of checking whether two sentences contain identical words, the application measures the distance between their embedding vectors.

This enables the AI engine to identify educational concepts even when different wording is used.

Applications include:

- Context Ranking
- Topic Detection
- Sentence Selection
- Information Retrieval

---

# 📚 KMeans Topic Clustering

After sentence embeddings have been generated, the application organizes similar educational concepts using the **KMeans Clustering Algorithm**.

Purpose of clustering:

- Group similar concepts
- Prevent repetitive questions
- Improve topic diversity
- Increase educational coverage

Example

```
Machine Learning
      │
      ├── Supervised Learning
      ├── Unsupervised Learning
      └── Reinforcement Learning
```

Instead of generating five questions from one paragraph, clustering distributes questions across multiple topics.
